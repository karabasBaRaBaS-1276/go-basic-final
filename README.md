# Структура базы данных
## Таблица для хранения списка задач (scheduler);
Поля таблицы:
- `id` - Уникальный идентификатор записи
- `date` - Дата задачи в формате YYYYMMDD
- `title` - Заголовок задачи
- `comment` - Комментарий к задаче
- `repeat` - Правило для повторений задачи. Формат:<br>
   **d <число>** - задача переносится на указанное число дней. Максимально допустимое число равно 400.<br>
   **y** - задача выполняется ежегодно. При выполнении задачи дата перенесётся на год вперёд.<br>
   **w <через запятую от 1 до 7>** - задача назначается в указанные дни недели, где 1 — понедельник, 7 — воскресенье.<br>
   **m <через запятую от 1 до 31, -1, -2> [через запятую от 1 до 12]** - задача назначается в указанные дни месяца. При этом вторая последовательность чисел опциональна и указывает на определённые месяцы.<br>

# Полезности для проверки реализации
## Сборка образа
```
docker build -t karabasbarabas1276/go-basic-final:v1.0.0 .
```
Если нужно пересобрать заново без кэша, то
```
docker build --no-cache -t karabasbarabas1276/go-basic-final:v1.0.0 .
```

## Запуск контейнера
Запустить контейнер с автоматическим удалением контейнера при выходе:
```
docker run -p 7540:7540 --rm karabasbarabas1276/go-basic-final:v1.0.0
```

Запустить контейнер с переопределением критических параметров окружения (рекомендуется):
```
docker run -e TODO_JWT_SECRET_KEY=secret-very-hard \
           -e TODO_PASSWORD=qwerty \
           -p 7540:7540 \
          --rm karabasbarabas1276/go-basic-final:v1.0.0
```
где в качестве значений `TODO_JWT_SECRET_KEY` и `TODO_PASSWORD` указать свои данные.

При запуске контейнера он в консоль выведет логи работы приложения.
